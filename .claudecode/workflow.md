# P25ML ワークフロー

## 新規シリーズ作成フロー

1. `series/_template/` をコピーして `series/SR{nn}_{slug}/` を作成
2. `series.yaml` を編集（シリーズID、タイトル、ジャンル、トーン、world_exposure等）
3. 主要キャラクターをローカル `docs/local_characters/` に作成
4. 必要に応じてグローバルキャラクターを `db/characters/` に作成
5. ローカル舞台を `docs/local_locations/` に作成
6. プロットアークを `plot/arcs/` に作成
7. `P25ML_MASTER.md` のシリーズ一覧を更新
8. `CHANGELOG.md` に記録

## 新規エピソード執筆フロー

### Phase 1: 準備
1. **検索ファースト**: 関連する既存設定を全検索
2. **執筆ノート参照**: `writing_notes.md` を読み、前回までの改善指示・蓄積された知見を確認する
3. **前話レビュー参照**: 直前のエピソードの `reviews/` ファイルがあ���ば、「次話への提案」を確認する

### Phase 2: 執筆
4. エピソードファイルを `episodes/` に作成（status: drafted）
5. 新規登場要素があれば `new_elements_introduced` に記録
6. 新規キャラクター・場所・組織はローカルdocsに登録
7. **設定フィードバック（ブロッキング）**: 執筆中に生まれた新しい設定は該当する設定ファイルに追記・新規作成する。設定ファイルへの反映内容をユーザーに報告すること。**設定フィードバックが完了するまでPhase 3（品質レビュー）に進んではならない。**
8. **設定フィードバック確認チェックリスト**（毎回必ず実行）:
   - [ ] 新規キャラクター・名前が登場していないか → `docs/local_characters/` に登録or追記
   - [ ] 新しい場所・施設が登場していないか → `docs/local_locations/` に登録or追記
   - [ ] 世界観のルール・仕様が具体化されていないか → `docs/local_settings/` に追記
   - [ ] 既存キャラの新しい側面（能力、関係性、過去）が描かれていないか → 該当キャラファイルに追記
   - [ ] 組織の新情報が出ていないか → 該当組織ファイルに追記
   - [ ] `SERIES_CATALOG.md` の「現在の状況」を更新する必要がないか
   - すべて「該当なし」または「反映済み」になるまでPhase 3に進まない
9. タイムラインとの整合性を確認

### Phase 3: 品質レビュー（執筆直後に必ず実施）
9. **自己レビュー**: 執筆したエピソードを読み返し、品質レビューを実施する（詳細は「品質レビュー規則」参照）
10. **レビューファイル作成**: `reviews/EP{NNN}_review.md` にレビュー結果を記録する
11. **判定**:
    - **A（合格）**: そのまま次のステップへ進む
    - **B（条件付き合格）**: 軽微な修正を行い、修正内容をレビューファイルに追記して次へ
    - **C（不合格・再執筆）**: 問題点を特定し、エピソードを再執��する。再執筆後、再度レビューを実施
12. **執筆ノート更新**: レビューで得られた知見のうち、**今後の執筆に継続的に活かせるもの**を `writing_notes.md` に追記する

### Phase 4: 確定
13. 執筆完了後、バリデーション実行
14. レビュー結果とともにユーザーに報告
15. ユーザー確認後に `status: revised` → `canonized` へ昇格

## 新規グローバルエンティティ作成フロー

1. **近似概念チェック**: 既存に類似エンティティがないか検索
2. IDを採番（連番、重複なし）
3. スキーマに従いファイル作成
4. 該当するグローバルインデックスを更新
5. `P25ML_MASTER.md` の該当一覧を更新
6. 必要に応じて `P25ML_TIMELINE.md` を更新
7. `CHANGELOG.md` に記録

## ローカルからグローバルへの昇格フロー

1. ローカルIDはそのまま保持
2. 新規グローバルIDを付与
3. `promotion_from_local` にローカルIDを記録
4. `db/` にグローバルファイルを作成
5. ローカルファイルに `promoted_to: CHAR-G-XXXX` 等を追記
6. グローバルインデックスを更新
7. `CHANGELOG.md` に記録

## 正史階層と運用

| 領域 | 正史レベル | 改変可否 |
|------|-----------|---------|
| `docs/` | 準正史 | 可（記録必須） |
| `plot/` | 計画 | 自由に変更可 |
| `episodes/` drafted | 草稿 | 自由に変更可 |
| `episodes/` revised | 改稿済 | 軽微修正のみ可 |
| `episodes/` canonized | **正史** | 重大修正禁止 |
| `episodes/` published | **公開正史** | 重大修正禁止 |

## 改稿時の記録

すべての改稿は `change_log.md` に以下の形式で記録すること。

```markdown
## YYYY-MM-DD — 変更タイトル

- **対象**: ファイルパス
- **種別**: 軽微修正 / retcon / 設定追加 / 設定変更
- **変更前**: 要約
- **変更後**: 要約
- **理由**: なぜ変更したか
- **影響範囲**: 他に影響するファイル・シリーズ
```

## クロスオーバー提案フロー

1. クロスオーバーの必要性を検出
2. 影響範囲を分析
3. **提案文書を作成**（自動実装しない）
4. ユーザーに提示
5. ユーザー承認後のみ実装
6. 両シリーズの `change_log.md` に記録
