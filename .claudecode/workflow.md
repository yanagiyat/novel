# P25ML ワークフロー

## 新規シリーズ作成フロー

1. `series/_template/` をコピーして `series/SR{nn}_{slug}/` を作成
2. `series.yaml` を編集（シリーズID、タイトル、ジャンル、トーン、world_exposure等）
3. 主要キャラクターをローカル `docs/local_characters/` に作成
4. 必要に応じてグローバルキャラクターを `db/characters/` に作成
5. ローカル舞台を `docs/local_locations/` に作成
6. プロットアークを `plot/arcs/` に作成
7. `P25ML_MASTER.md` のシリーズ一覧を更新
8. `CHANGELOG.md` に記録

## 新規エピソード執筆フロー

1. **検索ファースト**: 関連する既存設定を全検索
2. エピソードファイルを `episodes/` に作成（status: drafted）
3. 新規登場要素があれば `new_elements_introduced` に記録
4. 新規キャラクター・場所・組織はローカルdocsに登録
5. タイムラインとの整合性を確認
6. 執筆完了後、バリデーション実行
7. ユーザー確認後に `status: revised` → `canonized` へ昇格

## 新規グローバルエンティティ作成フロー

1. **近似概念チェック**: 既存に類似エンティティがないか検索
2. IDを採番（連番、重複なし）
3. スキーマに従いファイル作成
4. 該当するグローバルインデックスを更新
5. `P25ML_MASTER.md` の該当一覧を更新
6. 必要に応じて `P25ML_TIMELINE.md` を更新
7. `CHANGELOG.md` に記録

## ローカルからグローバルへの昇格フロー

1. ローカルIDはそのまま保持
2. 新規グローバルIDを付与
3. `promotion_from_local` にローカルIDを記録
4. `db/` にグローバルファイルを作成
5. ローカルファイルに `promoted_to: CHAR-G-XXXX` 等を追記
6. グローバルインデックスを更新
7. `CHANGELOG.md` に記録

## 正史階層と運用

| 領域 | 正史レベル | 改変可否 |
|------|-----------|---------|
| `docs/` | 準正史 | 可（記録必須） |
| `plot/` | 計画 | 自由に変更可 |
| `episodes/` drafted | 草稿 | 自由に変更可 |
| `episodes/` revised | 改稿済 | 軽微修正のみ可 |
| `episodes/` canonized | **正史** | 重大修正禁止 |
| `episodes/` published | **公開正史** | 重大修正禁止 |

## 改稿時の記録

すべての改稿は `change_log.md` に以下の形式で記録すること。

```markdown
## YYYY-MM-DD — 変更タイトル

- **対象**: ファイルパス
- **種別**: 軽微修正 / retcon / 設定追加 / 設定変更
- **変更前**: 要約
- **変更後**: 要約
- **理由**: なぜ変更したか
- **影響範囲**: 他に影響するファイル・シリーズ
```

## クロスオーバー提案フロー

1. クロスオーバーの必要性を検出
2. 影響範囲を分析
3. **提案文書を作成**（自動実装しない）
4. ユーザーに提示
5. ユーザー承認後のみ実装
6. 両シリーズの `change_log.md` に記録
