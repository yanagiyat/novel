# P25ML バリデーション規則

ファイル生成・更新後に以下の検査項目を自己点検すること。

## 検査項目一覧

### 1. YAML必須項目の欠落
- すべてのMarkdownファイルはYAMLフロントマターを持つこと
- 各スキーマで定義された必須フィールドが存在すること
- `id`, `name`, `type` は全エンティティで必須

### 2. 重複ID
- 同一カテゴリ内でIDが重複していないこと
- グローバルIDとローカルIDの名前空間が衝突していないこと

### 3. 壊れた参照ID
- `related_ids`, `affiliation`, `base_location` 等で参照されるIDが実在すること
- 参照先ファイルが存在すること

### 4. 日付矛盾
- `date_start` ≤ `date_end` であること
- イベント日付がプロジェクト起点 (2025-01-01) 以降であること（過去イベントを除く）
- タイムライン上の因果関係が成立すること

### 5. 正史ステータス矛盾
- `canon: true` のエンティティが `status: deprecated` でないこと
- `published` エピソードの `canon` が `true` であること
- `drafted` 段階のエピソードが `canon: true` になっていないこと

### 6. visibility整合性違反
- `author-only` のエンティティが `public` のエピソードで直接言及されていないこと
- visibility の値が定義済みの許容値であること（public / series-internal / author-only）

### 7. spoiler_scope違反
- `author-only` スコープの情報が `public` エピソードに含まれていないこと
- スコープの値が許容値であること（public / series-only / author-only）

### 8. 実在名の混入リスク
- 企業名、サービス名、SNS名、学校名、病院名、商業施設名が実在のものでないか
- 実在事件に酷似した設定が含まれていないか

### 9. 近似既存設定の重複生成
- 新規作成エンティティが既存エンティティと機能的に重複していないか
- 名称が類似する既存エンティティがないか

### 10. グローバルとローカルの誤配置
- グローバルID (`*-G-*`) を持つエンティティが `db/` に配置されていること
- ローカルID (`*-SRXX-*`) を持つエンティティが対応シリーズの `docs/` に配置されていること

### 11. クロスオーバー規則違反
- 別シリーズのキャラクターが無断で登場していないこと
- クロスオーバーにユーザー確認記録があること

### 12. 既存正史の無断改変
- `canonized` / `published` 済みのエピソード本文が変更されていないこと
- 正史設定が無断で書き換えられていないこと

### 13. deleted扱いの不正運用
- ファイル削除ではなく `deprecated` 化で対応しているか
- 削除が行われた場合、正当な理由と記録があるか

## 検査結果の出力形式

```markdown
## バリデーション結果 — YYYY-MM-DD

### ✅ 合格項目
- [ ] 項目名: 説明

### ⚠️ 警告項目
- [ ] 項目名: 説明と対処案

### ❌ 不合格項目
- [ ] 項目名: 説明と必要な修正
```
