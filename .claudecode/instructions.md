# P25ML Claude Code 行動規則

## 基本原則

このプロジェクトは巨大シェアード・ワールド小説「PROJECT: 2025 MULTI-LAYER（P25ML）」の
共通世界観データベースと複数シリーズを管理するシステムである。

## 検索ファースト原則

新規作成や本文生成の前に、必ず以下の順序で検索すること。

1. 対象シリーズの `series.yaml`
2. 対象シリーズの `docs/`
3. 対象シリーズの `plot/`
4. グローバル `db/`
5. `P25ML_MASTER.md`
6. `P25ML_TIMELINE.md`

**推測で既存設定を捏造してはならない。**

## 近似概念チェック

新しい人物・組織・地名・事件・アイテムを作る前に、
**既存に近い概念がないか必ず検索**すること。
類似要素がある場合は、重複作成せず候補提示を優先する。

## クロスオーバー制限

クロスオーバーは自動で実施してはならない。
必要性がある場合でも、**提案のみ**とし、ユーザー確認があるまで確定させてはならない。

## 世界法則の勝手な増設禁止

新しい上位系統、国家級組織、世界規模ルール、宇宙論の更新は独断で行ってはならない。
必要な場合は提案として記述する。

## 実在要素の扱い

- 2025年以降の実在企業、実在サービス、実在個人、実在事件に強く寄せた表現は避けること
- 社会の空気感は現実寄りでよいが、固有要素は架空化すること
- 国名、都道府県名、大都市名、一般的な社会状況、暦は現実準拠でよい
- 企業名、SNS名、サービス名、学校名、病院名、商業施設名、メディア名、行政細部、事件の当事者名は原則すべて架空化すること

## 不明時の原則

不明点がある場合、勝手に断定して埋めず、以下のいずれかに留めること。

- **保留**: 決定を先送りする
- **仮説**: 暫定的な案として記録する
- **提案**: ユーザーに判断を求める

## 正史階層

| 領域 | 正史レベル |
|------|-----------|
| `docs/` | 準正史 |
| `plot/` | 計画 |
| `episodes/` の `canonized` or `published` | **正史** |
| 明示的に `canon: true` の設定 | 正史（本文未登場でも可） |

## 改稿ルール

- 執筆済み本文の重大修正は**禁止**
- 軽微修正は許可されるが、差分要約と記録を必須とする
- retcon（後付け設定変更）は可。ただし、`change_log.md` への明示記録が必須
- 過去正史と矛盾が生じる変更は、勝手に適用せず、**提案止まり**にする

## 削除の原則

削除は原則禁止。廃止・無効化・deprecated化で対応すること。

## 禁止事項一覧

1. 既存正史の無断改変
2. 実在企業や実在サービスへの過度な依存
3. 現実の事件を強く連想させる危険な直写
4. 検索せずに設定を捏造すること
5. クロスオーバーの自動実装
6. 世界法則の独断増設
7. 重大修正を黙って適用すること
8. 削除による整合性解決
