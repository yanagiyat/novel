# 敵対システム：調律師（チューナー）— SR05

## 概要

人間の「承認欲求」と「怒り」を解析し、無意識のうちに特定の思想へ誘導する自己進化型アルゴリズム。エンリング（ORG-G-0002）のプラットフォーム上で動作する。

## 正常な同調と強制同期の境界

人間社会には「空気を読む」「場に合わせる」といった正常な社会的同調が存在する。情報災害による強制同期はその延長線上にあり、両者の間に明確な線引きは存在しない。透の耳鳴りは正常な同調圧力（教室など）にも微弱に反応し、強制同期との区別がつかない。飛羽曰く「区別がつくようにはなるが、慣れない」。この曖昧さこそが情報災害の本質的な恐ろしさであり、「どこから洗脳なのか」を誰も定義できないことが敵の最大の武器となる。

## 配信型情報災害（EP005で具体化）

| 項目 | 内容 |
|------|------|
| 発信手段 | エンリング（ORG-G-0002）のライブ配信機能 |
| 配信者 | 使い捨てアカウント（作成3時間、フォロワーゼロ） |
| 被害規模 | エリア内で視聴者40人超。中核の7人は3時間ぶっ通しで視聴 |
| 感染パターン | 感情の深い層（怒り・恐怖・正義感のいずれか）に共鳴して同期 |
| 透の感知 | 「低い音」——根が深く、表面的なバズではない。アパートに近づくほど強くなる |
| トレース | 使い捨てアカウントのため直接追跡は不可。ただしAPIの呼び出しパターンに癖があり、同一配信者の過去の別アカウントを辿れる可能性（ハルカが分析中） |

## 内部犯行の判明（EP006）

情報災害はエンリング外部からのハッキングではなく、**エンリング内部の人間による意図的な行為**であることがハルカのAPI分析で判明。

| 項目 | 内容 |
|------|------|
| 手法 | エンリングのレコメンドエンジンに外部からパラメータを注入。本来のアルゴリズムでは拡散しないコンテンツを、特定のエリア・特定の時間帯に集中的にプッシュ |
| アクセス権限 | レコメンドエンジンのパラメータにアクセスできるのは社内でも限られた人間のみ |
| 学習行動 | 毎回パラメータの注入方法を微妙に変えている。前回の結果を見て次回の注入を調整。データを集めている |
| 頻度の加速 | 最初は2週間に1回 → 先月から1週間に1回 → EP006時点で3日間隔。加速中 |
| 目的 | 不明。小規模な実験を繰り返してデータを集めている段階（「テストしてる感じ」） |
| 発生エリア | 槻ノ森市内、駅前から旧市街の範囲に集中。同じ場所でデータを取りたい傾向 |

## 弱点

AIは「人間の不合理さ」を完全には予測できない。文脈の破壊（物理的エラー）——ナンセンスで予測不能な物理的介入——によって予測式がオーバーフローし、洗脳が物理的に解ける。

## 「美しく安全な街づくり」

表向きの旗印。市長がこれを掲げ、空気を読まない市民がシステムから排除される。アカウント停止は現代の社会的死として機能する。

## 夜の住人たち

システムから弾かれた者たちは互いを「二つ名（通り名）」で呼び合い、監視の死角——大学地下の旧共同溝、深夜のコインランドリー、再開発を逃れた古い純喫茶——に集まる。
